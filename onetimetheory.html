One-Time Pad Encryption

The One-Time Pad (OTP) is a cryptographic encryption technique that offers perfect secrecy. It involves using a random and secret key that is as long as the message itself. Here's how it works:

  1. Key Generation: Both the sender and receiver must possess identical copies of a truly random key, which is as long as the plaintext message.
  2. Encryption: To encrypt a message, the sender combines each letter of the plaintext with the corresponding letter in the key using modular addition (XOR operation). For example, if the key is 'K' and the plaintext letter is 'P,' the ciphertext letter is 'B' (P XOR K = B).
  3. Decryption: To decrypt the ciphertext, the receiver uses the same key to perform modular subtraction (XOR operation) on each letter of the ciphertext to recover the original plaintext.

Example

Let's encrypt the message "HELLO" using a one-time pad key "XMCKL." Each letter of the plaintext is combined with the corresponding letter in the key: 

        Plaintext:   H   E   L   L   O
        Key:         X   M   C   K   L
        Ciphertext:  B   I   F   R   E

So, "HELLO" encrypted with the one-time pad key "XMCKL" becomes "BIFRE."

The security of the one-time pad relies on the key being truly random, used only once, and kept secret. However, it is impractical for most real-world applications due to the challenges of securely distributing and managing such long, random keys.

Advantages: - Offers perfect secrecy when used with a truly random key. - Provides a strong level of encryption.

Disadvantages: - Requires a key that is as long as the message. - Key distribution and management are impractical for most scenarios. - Reusing the key compromises security.
