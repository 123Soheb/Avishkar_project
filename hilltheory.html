Hill Cipher Definition

The Hill Cipher is a polygraphic substitution cipher based on linear algebra. It was invented by Lester S. Hill in 1929. Unlike simple substitution ciphers, the Hill Cipher encrypts blocks of text at a time, making it more secure.

Here's how the Hill Cipher works:

  1. Key: The Hill Cipher uses a matrix as its encryption key. This matrix is typically a square matrix with a size equal to the length of the block of text to be encrypted. The key matrix must be invertible.
  2. Encryption: To encrypt a block of text, it is first converted into numerical values, usually using a scheme like assigning 'A' as 0, 'B' as 1, and so on. The block is then multiplied by the key matrix modulo the size of the alphabet (e.g., modulo 26 for the English alphabet).
  3. Decryption: Decryption involves multiplying the ciphertext by the modular inverse of the key matrix, resulting in the original plaintext block.

The Hill Cipher provides a higher level of security compared to simple substitution ciphers, but it requires a key matrix and mathematical operations that might not be easy to perform by hand.

Example:

Let's illustrate the Hill Cipher with an example using a 2x2 key matrix and a block of text. We'll use the key matrix: 

        | 6  24 |
        | 13 16 |

And we want to encrypt the plaintext block "HELLO." First, we convert the letters to numerical values ('H' as 7, 'E' as 4, 'L' as 11, and 'O' as 14).

Now, we perform the encryption:

        H => 7    E => 4    L => 11   L => 11   O => 14

Now, we group these numerical values into a 2x1 matrix:

        | 7 |
        | 4 |

Next, we multiply this matrix by our key matrix:

        | 6  24 |   | 7 |   =   | 170 |
        | 13 16 |   | 4 |       | 152 |

Now, we reduce these values modulo 26:

        170 % 26 = 18    (R)
        152 % 26 = 24    (X)

So, the encrypted block "HELLO" becomes "RX" using the Hill Cipher with the given key matrix.
