Lomuto Sort Algorithm

Lomuto Sort is a sorting algorithm often associated with the Lomuto partition scheme in the Quick Sort algorithm. It's known for its simplicity and ease of implementation. While it may not be as efficient as some other sorting algorithms, it's still a valuable concept to understand.

        Algorithm
        Lomuto Sort is a straightforward sorting algorithm that operates by repeatedly selecting a "pivot" element and partitioning the array into two subarrays: elements less than the pivot and elements greater than or equal to the pivot. The subarrays are then recursively sorted.

        Here's a simplified overview of the algorithm:

        1. **Choose a Pivot**: Select a pivot element from the array. The choice of pivot can impact the algorithm's performance.

        2. **Partitioning**: Rearrange the elements in the array so that all elements less than the pivot come before all elements greater than or equal to the pivot. This process is called partitioning.

        3. **Recursive Sorting**: Recursively apply the Lomuto Sort algorithm to the subarrays created by the partitioning step.

        begin LomutoSort(arr, low, high)  
           if low < high
               // Partition the array and get the pivot index
               pivotIndex = LomutoPartition(arr, low, high)

               // Recursively sort the subarrays
               LomutoSort(arr, low, pivotIndex - 1)
               LomutoSort(arr, pivotIndex + 1, high)
           end if
        end LomutoSort

        begin LomutoPartition(arr, low, high)  
           pivot = arr[high]
           i = low - 1

           loop
               repeat
                   i = i + 1
               until arr[i] >= pivot

               if i < high
                   swap arr[i] and arr[j]
               end if
           end loop

           // Swap the pivot into its correct position
           swap arr[i] and arr[high]

           return i
        end LomutoPartition

    Working of Lomuto Sort

     
        Lomuto Sort is a straightforward sorting algorithm that operates by repeatedly selecting a "pivot" element and partitioning the array into two subarrays: elements less than the pivot and elements greater than or equal to the pivot. Here's a detailed explanation of how it works:

        1. **Choose a Pivot**: The algorithm starts by selecting a pivot element from the array. The choice of pivot can impact the algorithm's performance.

        2. **Partitioning**: The array is rearranged so that all elements less than the pivot come before all elements greater than or equal to the pivot. This partitioning is achieved using a loop and two pointers: `i` and `j`. `i` finds the first element greater than or equal to the pivot, and `j` tracks the current element being considered. When `i` finds such an element, it swaps it with `j`, ensuring that elements less than the pivot are on the left side, and those greater or equal are on the right side.

        3. **Recursive Sorting**: After the partitioning step, the pivot is in its correct position. The algorithm then recursively sorts the subarrays to the left and right of the pivot, using the same Lomuto Sort algorithm.

        Lomuto Sort is easy to implement but may not be as efficient as some other sorting algorithms. However, it's a useful concept to understand, particularly in the context of Quick Sort and partitioning schemes.
