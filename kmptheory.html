Knuth-Morris-Pratt (KMP) Algorithm

In this article, we will explore the Knuth-Morris-Pratt (KMP) Algorithm, a powerful string searching algorithm used to find occurrences of a pattern within a text. The KMP Algorithm is known for its efficiency and ability to handle large texts, making it a valuable tool in text processing, string matching, and data retrieval applications.

        Algorithm
        The Knuth-Morris-Pratt (KMP) Algorithm is an efficient string searching algorithm that utilizes a precomputed "failure" or "partial match" table to skip unnecessary comparisons. Here's a high-level summary of the algorithm:

        1. **Create a Partial Match Table**: Before searching for the pattern in the text, create a Partial Match Table (also known as the LPS - Longest Prefix Suffix) that stores information about the longest proper prefix that is also a suffix for each substring of the pattern.

        2. **Search for the Pattern in the Text**: Start searching for the pattern within the text using two pointers, one for the text and one for the pattern. Compare characters from the text and the pattern while considering the information from the Partial Match Table.

        3. **Partial Match Table Usage**: When a mismatch occurs between a character in the pattern and the corresponding character in the text, use the Partial Match Table to determine how many characters in the pattern can be skipped in the next comparison. This skips unnecessary comparisons and improves the efficiency of the algorithm.

        begin KMPSearch(text, pattern)  
           // Create the Partial Match Table for the pattern
           lps = ComputeLPSTable(pattern)

           textLength = length of text
           patternLength = length of pattern
           textIndex = 0
           patternIndex = 0

           while textIndex < textLength
               if pattern[patternIndex] = text[textIndex]
                   increment both patternIndex and textIndex
               end if

               if patternIndex = patternLength
                   // Pattern found, return its starting index in the text
                   return textIndex - patternIndex
               end if

               else if textIndex < textLength and pattern[patternIndex] != text[textIndex]
                   if patternIndex != 0
                       // Use the Partial Match Table to skip characters in the pattern
                       patternIndex = lps[patternIndex - 1]
                   else
                       increment textIndex
                   end if
               end else
           end while

           // Pattern not found in the text
           return -1
        end KMPSearch

    Working of KMP Algorithm

     
        The Knuth-Morris-Pratt (KMP) Algorithm efficiently searches for a pattern within a text using a precomputed Partial Match Table. Here's a detailed explanation of how it works:

        1. **Create a Partial Match Table**: Before searching, create a Partial Match Table for the pattern. This table stores information about the longest proper prefix that is also a suffix for each substring of the pattern. It helps skip unnecessary comparisons during the search.

        2. **Search for the Pattern in the Text**: Start searching for the pattern within the text using two pointers, one for the text and one for the pattern. Compare characters from the text and the pattern while considering the information from the Partial Match Table.

        3. **Partial Match Table Usage**: When a mismatch occurs between a character in the pattern and the corresponding character in the text, use the Partial Match Table to determine how many characters in the pattern can be skipped in the next comparison. This efficient skipping of characters enhances the algorithm's speed.

        The KMP Algorithm continues this process until it either finds the pattern in the text (returning its starting index in the text) or exhausts all possibilities without a match (returning -1 to indicate that the pattern is not present).

        KMP's efficiency and ability to handle large texts make it a valuable tool for tasks like searching, data retrieval, and text processing.
