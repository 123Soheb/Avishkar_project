Columnar Transposition Cipher

The Columnar Transposition Cipher is a transposition cipher that rearranges the order of the letters in the plaintext according to a certain system. Here's how it works:

  1. Key: The key is a secret word or phrase that determines the arrangement of the columns. The unique letters in the key are written in alphabetical order to create the key.
  2. Encryption: To encrypt a message, write the plaintext in rows under the columns determined by the key. If there are extra spaces in the last row, fill them with any character (often 'X'). Then, read the ciphertext column by column, starting from the column with the first letter of the key, and so on.
  3. Decryption: To decrypt the ciphertext, the recipient uses the same key to determine the order of the columns. Then, the ciphertext is written in rows under the columns, and the plaintext is read row by row.

Example

Let's encrypt the message "HELLO" using the key "SECRET." First, we arrange the columns based on the alphabetical order of the unique letters in the key: 

        Key:   C E R S T
        ----------------
        Text:  H E L L O

Now, we read the ciphertext column by column:

        C E R S T
        ---------
        H E L L O

So, "HELLO" encrypted using the Columnar Transposition Cipher with the key "SECRET" becomes "HOELL."

The security of this cipher depends on the key, and it is important to keep the key secret. Repeating characters in the key are given a unique label (e.g., "SECRTO" would become "SECRTOX" with 'X' as a filler).

Advantages: - Provides a level of security. - Requires knowledge of the key to decrypt.

Disadvantages: - Vulnerable to frequency analysis. - Security depends on the key and its secrecy.
